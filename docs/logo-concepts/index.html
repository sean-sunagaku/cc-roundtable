Meeting Room ロゴ案

今回は「もっとシンプルに、でも実際のアプリの様子を表す」という条件に寄せて、 MeetingScreen の見た目から記号を抜き出しました。 具体的には、上部のステータス pills、中央のチャット bubbles、下部の入力や terminal prompt をロゴ化しています。 配色は、明るいベースと濃いインクの完全な 2 色に絞っています。

UI Card

一番バランスが良い

実際のアプリ画面を最小限まで削った案です。上部の agent/status pills、中央のメッセージ bubbles、下部の prompt bar が入っています。

実アプリとの対応

上段は status-row と接続状態、中央は ChatView、下段は InputBar と terminal の送信感です。

Prompt Bubble

最もシンプル

チャット bubble と terminal prompt をひとつに畳み込んだ記号です。アプリアイコンや小さい表示サイズで最も潰れにくいです。

実アプリとの対応

会議の「メッセージ送信」と Claude の「prompt 入力」を一体化した見せ方です。余計なモチーフをほぼ消しています。

Split View

画面構成を最も忠実に反映

左のチャット領域と右の terminal 領域をそのまま抽象化した案です。プロダクトの構造が最も伝わりますが、少しだけ説明的です。

実アプリとの対応

ChatView と TerminalPane の共存が分かりやすいです。プロダクト説明用のロゴとしては相性が良いです。
