# cc-roundtable 競合トンマナ調査レポート

## 調査日: 2026-03-06

---

## 1. 調査対象の選定理由

cc-roundtable は「AI エージェント間の議論をリアルタイムで可視化するデスクトップアプリ」であり、以下の3つの軸で競合・参考サービスを選定した。

| 軸 | 選定理由 |
|---|---|
| AI エージェント可視化 | 直接競合。マルチエージェント会話の見せ方 |
| 開発者向けモダン UI | ターゲットユーザーが日常使うツールのデザイン言語 |
| AI チャットインターフェース | チャットベース UI のベンチマーク |

選定した5社: **LangGraph Studio**, **CrewAI**, **AutoGen Studio**, **Cursor**, **Warp Terminal**

## 3. 競合ポジショニングマップ

### 軸の定義
- **X軸**: テクニカル（左）<---> アプローチャブル（右）
- **Y軸**: ミニマル（下）<---> リッチ（上）

> **注**: 当初「フレンドリー」としていた X 軸右側を「アプローチャブル」に修正。brand-strategist との議論により、「フレンドリー」は「簡易版/初心者向け」を連想させるリスクがあり、cc-roundtable が狙うのは「approachable but authoritative（親しみやすいが権威ある）」ポジションであるため。

```
リッチ (情報密度高い / 装飾的)
    ^
    |
    |   AutoGen Studio        CrewAI
    |      [研究的/多機能]       [活気/Enterprise]
    |
    |
    |   LangGraph Studio
    |      [構造的/ノードグラフ]
    |
    |
    |   Warp
    |      [モダン/カスタマイズ]
    |
    |                          ★ cc-roundtable
    |                            (approachable authority)
    |
    |   Cursor
    |      [洗練/ダーク]
    |
    +------------------------------------------>
テクニカル                          アプローチャブル
(専門的/複雑)                     (直感的/権威ある親しみ)
```

### 各社ポジション解説

| 競合 | X軸 (テクニカル←→アプローチャブル) | Y軸 (ミニマル←→リッチ) |
|---|---|---|
| LangGraph Studio | テクニカル寄り (ノードグラフ, デバッグ中心) | 中程度 (2ペイン, グラフ表示) |
| CrewAI | やや アプローチャブル (ドラッグ&ドロップ, ビジュアルエディタ) | リッチ寄り (多機能UI, Enterprise機能) |
| AutoGen Studio | テクニカル寄り (Research発, JSON設定) | リッチ (3セクション, フロー可視化) |
| Cursor | テクニカル寄り (VS Code ベース, 開発者向け) | ミニマル (クリーン, 黒背景, 最小限のUI) |
| Warp | やや テクニカル (ターミナル) | やや ミニマル (ブロックベース, カスタマイズ) |

## 4. デザイントレンド分析

### 共通パターン
1. **ダークモードがデフォルト**: 5社中4社がダーク背景をデフォルト採用（AutoGen Studio のみライト寄り）
2. **サンセリフ統一**: 全社がサンセリフフォントを採用。モノスペースはコード/ターミナル表示のみ
3. **ブルー系アクセント多用**: Cursor (#228DF2), Warp (#268BD2) ともにブルー。LangChain はグリーン系で差別化
4. **マルチペインレイアウト**: サイドバー + メインコンテンツ + コンテキストパネルの3列構成が主流
5. **低彩度背景 + 高彩度アクセント**: 情報密度が高いため、背景は抑え、アクセントで注意を引く手法

### 差別化要素
| 競合 | 差別化カラー |
|---|---|
| LangGraph | ターコイズ/エメラルドグリーン |
| CrewAI | コーラルレッド (#FF5A50) |
| AutoGen | ライトテーマ + Microsoft ブルー |
| Cursor | ディープブラック + パステルブルー |
| Warp | パープル/ブルーグラデーション |

## 5. 空いているポジションと差別化機会

### 5.1 空いているポジション

現在の競合マップにおいて、以下のポジションが手薄である。

**「アプローチャブル・オーソリティ x やや ミニマル」ゾーン**

- ほとんどの競合がテクニカル寄りに位置している
- エージェント会話の「観察・参加」という cc-roundtable の UX は、開発ツールというよりも「会議室に参加する」体験に近い
- この「会議室メタファー」を活かした、権威がありつつも親しみやすく、クリーンな UI は現状どの競合もカバーしていない
- 「Approachable but authoritative」= 簡易化ではなく、高い能力を親しみやすい形で提示する姿勢

### 5.2 cc-roundtable への提案

| 提案項目 | 内容 | 根拠 |
|---|---|---|
| **ポジション** | 「アプローチャブル・オーソリティ x やや ミニマル」 | 競合の空白地帯。会議室メタファー + Sage アーキタイプとの整合性 |
| **カラー戦略** | ダーク背景（開発者の期待に合致） + バーニッシュドゴールド (#C9A96E ~ #B8975A 付近) をブランドアクセントに | ブルー系は Cursor/Warp と衝突。グリーンは LangChain 領域。コーラルは CrewAI。アンバー/オレンジは warning convention と衝突。ゴールドは「書斎の間接照明/図書館の金箔」-- Sage/Scholarly メタファーと直接整合し、競合カラーマップ上で完全に空いている |
| **フォント** | Serif（英語見出し/エージェント名）+ ゴシック体（日本語、上品なウェイト）+ Sans-serif（UI chrome）+ Monospace（コード） | 5社全てがサンセリフ -- Serif は最大の差別化要素。ただし日本語明朝体は「文芸誌」印象が強すぎるため、言語別に最適化。Serif+Sans ハイブリッドは「Scholarly but alive」テンションをタイポグラフィで直接体現 |
| **レイアウト密度** | 中密度。チャット中心だがエージェント状態の可視化を控えめに統合 | 高密度（AutoGen/CrewAI）と低密度（Cursor）の中間。リアルタイム会話の流れを邪魔しない。参照モデル: Linear の Issue 詳細ビュー（構造化されているが読みやすい） |
| **トーン** | 「Scholarly but alive（学究的だが生きている）」 | brand-strategist のコアテンションと整合。競合が「ビルダー」「IDE」「デバッガ」に位置する中、「知的な観察空間」は唯一無二 |

### 5.3 避けるべきポジション

- **テクニカル x リッチ**: AutoGen Studio / CrewAI の領域。機能過多になりやすい
- **テクニカル x ミニマル**: Cursor の領域。コードエディタ文脈でのみ成立する美学
- **ブルー系アクセント**: 飽和状態。差別化が困難
- **アンバー/オレンジ系アクセント**: 開発者ツールにおける warning convention との衝突リスク

### 5.4 差別化キーワード

cc-roundtable のユニークな立ち位置を一言で表現するなら:

> **「エージェントの議論が見える会議室」**
> -- 開発ツールではなく、知的な観察・参加空間としての AI エージェント体験

競合が「ビルダー/IDE/デバッガ」として位置付けるのに対し、cc-roundtable は「scholarly observer / participant（学究的な観察者/参加者）」としての体験を提供する。この根本的な UX の違いをトンマナに反映させることが最大の差別化機会である。

### 5.5 競合アーキタイプ比較（Round 1 議論を反映）

| 競合 | 暗黙的アーキタイプ | cc-roundtable との距離 |
|---|---|---|
| LangGraph Studio | Creator/Sage（実験室の研究者） | 近い -- ただし LangGraph は「作る側」、cc-roundtable は「理解する側」 |
| CrewAI | Explorer/Hero（自律チームの力） | 遠い -- エネルギッシュ vs 落ち着き |
| AutoGen Studio | Sage（アカデミック/リサーチ） | 最も近い -- ただし「研究ラボの Sage」vs「会議室の Sage」で差別化 |
| Cursor | Magician/Creator（変容の約束） | 遠い -- 「最高のツール」vs「最高の観察体験」 |
| Warp | Explorer/Creator（再発明） | 中程度 -- モダンさは共有するがメタファーが異なる |
