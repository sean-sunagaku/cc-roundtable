# cc-roundtable - Logo Design Context

## Brand Information
- **App Name**: cc-roundtable
- **Overview**: Multi-agent discussion service for local codebases. Multiple AI agents gather in a virtual "roundtable" meeting to discuss, investigate, and implement code collaboratively. Humans can join and observe in real-time.
- **Target**: Developers (high technical level), AI/ML practitioners
- **Usage**: App icon (Electron desktop app) + brand logo
- **Platform**: Electron desktop + browser UI

## Key Concepts
- **Roundtable**: Agents sit around a virtual table to discuss — the central metaphor
- **Multi-agent collaboration**: Multiple AI agents with different specialties (researcher, engineer, PM, etc.)
- **Code-centric**: All discussions center around a local codebase
- **Meeting room**: A shared space where agents and humans discuss together
- **Real-time**: Live chat, terminal view, event streaming

## Design Direction
- Professional, developer-tool aesthetic
- Should feel like a serious development tool, not a toy
- Clean, modern, slightly technical
- Should work at small sizes (app icon) and large (brand)

## Constraints & Preferences
- Must work as macOS/Windows app icon (rounded square context)
- Needs to be recognizable at 16x16px
- "cc" prefix suggests "Claude Code" connection
- No specific color preference yet — derive from the product personality
- Avoid: overly playful/cartoon, generic chat bubble, generic AI brain imagery

## Competitive Context
- Claude Code (CLI tool by Anthropic)
- Cursor, Windsurf (AI coding tools)
- Slack, Discord (chat/collaboration tools)
- Linear, Notion (dev productivity tools)

## Accumulated Insights

### Step 1 Learnings
- "Roundtable" alone is not enough — user wants "ROOM" feeling (enclosed space)
- Abstract circles/rings feel like network diagrams, not spaces
- Pure code symbols and lettermarks lack spatial quality
- Top-down floor-plan view (V5) was closest to what user wants
- Key insight: The logo should convey "a space you enter to discuss" not just "nodes in a circle"
- Direction for Step 2: Room/enclosure + roundtable inside. Every concept needs walls/boundary/threshold

### Step 2 Learnings
- Overlapping circles/rings concept (V3 Ring Overlap) was the most promising direction
- User liked the idea of "overlapping roundtables where the overlap = the room"

### Step 3 Learnings (CRITICAL)
- Adding Claude elements (sparkles/dots) as decoration DOES NOT WORK — they feel meaningless and disconnected
- Every single element must have a clear "why" — no decorative additions
- The elements must be INTEGRATED into one unified symbol, not assembled as separate parts
- "まとまりがない" (no cohesion) = biggest problem. Must feel like ONE thing, not a collection
- Need much higher design quality/sophistication — current output feels amateur
- Key question: How to make "multiple Claudes forming a room" into ONE elegant symbol?
